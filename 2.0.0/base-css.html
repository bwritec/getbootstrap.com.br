Títulos & cópia de body

Escala tipográfica

Toda a régua tipográfica é baseada em duas variáveis Less em nosso arquivo variables.less: @baseFontSize e @baseLineHeight. O primeiro é o tamanho da fonte base usado e o segundo é a altura da linha base.

Usamos essas variáveis, e um pouco de matemática, para criar as margens, preenchimentos e alturas de linha de todos os nossos tipos e muito mais.

Código Snippets de código inline e de bloco

Inline

Envolva trechos de código embutidos com <code>. 

Por exemplo, <code>section</code> deve ser agrupado como inline.

Bloco básico

Use <pre> para várias linhas de código. Certifique-se de transformar todos os sinais de intercalação em caracteres Unicode para uma renderização adequada. 

<p>
    Exemplo de texto aqui...
</p>
<pre>
    &lt;p&gt;
        Exemplo de texto aqui...
    &lt;/p&gt;
</pre>

Observação: Certifique-se de manter o código dentro das tags <pre> o mais próximo possível da esquerda; ele renderizará todas as guias.

Google Prettify

Pegue o mesmo elemento <pre> e adicione duas classes opcionais para renderização aprimorada. 

<p>Exemplo de texto aqui...</p>

<pre class="prettyprint linenums">
  &lt;p&gt;Exemplo de texto aqui...&lt;/p&gt;
</pre>

Baixar google-code-prettify e veja o leia-me sobre como usar.

Formulários

HTML e CSS flexíveis

A melhor parte dos formulários no Bootstrap é que todas as suas entradas e controles ficam ótimos, não importa como você os cria em sua marcação. Nenhum HTML supérfluo é necessário, mas fornecemos os padrões para quem precisa.

Layouts mais complicados vêm com classes sucintas e escalonáveis para fácil estilização e vinculação de eventos, para que você esteja coberto em cada etapa.

Quatro layouts incluídos

Bootstrap vem com suporte para quatro tipos de layouts de formulário:

  • Vertical (padrão)
  • Busca
  • Inline
  • Horizontal

Diferentes tipos de layouts de formulário exigem algumas alterações na marcação, mas os próprios controles permanecem e se comportam da mesma forma.

Estados de controle e muito mais

Os formulários do Bootstrap incluem estilos para todos os controles básicos do formulário, como entrada, área de texto e seleção esperada. Mas também vem com vários componentes personalizados, como entradas anexadas e pré-anexadas e suporte para listas de caixas de seleção.

Estados como erro, aviso e sucesso são incluídos para cada tipo de controle de formulário. Também estão incluídos estilos para controles desabilitados.

Quatro tipos de formulários

Bootstrap fornece marcação e estilos simples para quatro estilos de formulários web comuns.

Anexar & anexar entradas

Grupos de entrada - com texto anexado ou pré-anexado - fornecem uma maneira fácil de fornecer mais contexto para suas entradas. Ótimos exemplos incluem o sinal @ para nomes de usuário da conta ou $ para finanças.

Checkbox e radios

Até a versão 1.4, o Bootstrap exigia marcação extra em torno das caixas de seleção e dos rádios para empilhá-los. Agora, é uma simples questão de repetir o <label class="checkbox"> que envolve o <input type="checkbox">.

Caixas de seleção e rádios inline também são suportados. Basta adicionar .inline para qualquer .checkbox ou .radio e pronto.

Formulários inline e append/prepend

Para usar entradas pré-anexadas ou anexadas em um formulário inline, certifique-se de colocar o .add-on e input na mesma linha, sem espaços.

Texto de ajuda do formulário

Para adicionar texto de ajuda às entradas do formulário, inclua texto de ajuda embutido com <span class="help-inline"> ou um bloco de texto de ajuda com <p class="help-block"> após o elemento de entrada.

Botões para ações

Como convenção, os botões só devem ser usados para ações, enquanto os hiperlinks devem ser usados para objetos. Por exemplo, "Baixar" deve ser um botão, enquanto "atividade recente" deve ser um link.

Para links e formulários

Os estilos de botão podem ser aplicados a qualquer coisa com o .btn aplicado. No entanto, normalmente você desejará aplicá-los apenas a elementos <a> e <button>.

Observação: Todos os botões devem incluir a classe .btn. Os estilos de botão devem ser aplicados a elementos <button> e <a> para consistência.

Compatibilidade entre navegadores

No IE9, eliminamos o gradiente em todos os botões em favor dos cantos arredondados, já que o IE9 não remove gradientes de fundo nos cantos.

Ícones Graciously fornecido por Glyphicons

Heads up ! As classes de ícones são ecoadas via CSS :after. Nos documentos, mostramos uma cor de fundo vermelho claro ao passar o mouse para destacar o tamanho do ícone.

Construído como um sprite

Em vez de tornar cada ícone uma solicitação extra, nós os compilamos em um sprite - um monte de imagens em um arquivo que usa CSS para posicionar as imagens com background-position. Este é o mesmo método que usamos no bwrite.com.br e funcionou bem para nós.

Todas as classes de ícones são prefixadas com .icon- para namespace e escopo adequados, assim como nossos outros componentes. Isso ajudará a evitar conflitos com outras ferramentas.

Glyphicons nos concedeu o uso do conjunto Halflings em nosso kit de ferramentas de código aberto, desde que forneçamos um link e crédito aqui nos documentos. Por favor, considere fazer o mesmo em seus projetos.

Como usar

Com v2.0.0, optamos por usar uma tag <i> para todos os nossos ícones, mas eles não têm nenhuma classe de caso - apenas um prefixo compartilhado. Para usar, coloque o seguinte código em qualquer lugar: 

<i class="icon-search"></i>

Também existem estilos disponíveis para ícones invertidos (brancos), preparados com uma classe extra: 

<i class="icon-search icon-white"></i>

Existem 120 classes para escolher para seus ícones. Basta adicionar uma tag <i> com as aulas certas e pronto. Você pode encontrar a lista completa em sprites.less ou aqui mesmo neste documento.

Casos de uso

Os ícones são ótimos, mas onde usá-los ? Aqui estão algumas ideias:

  • Como recursos visuais para a navegação da barra lateral.
  • Para uma navegação puramente baseada em ícones.
  • Para que os botões ajudem a transmitir o significado de uma ação.
  • Com links para compartilhar contexto no destino de um usuário.

Essencialmente, em qualquer lugar onde você possa colocar uma tag <i>, você pode colocar um ícone.

Exemplos

Use-os em botões, grupos de botões para uma barra de ferramentas, navegação ou entradas de formulário anexadas.
