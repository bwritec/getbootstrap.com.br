Títulos e cópia de body

Escala tipográfica

Toda a régua tipográfica é baseada em duas variáveis less em nosso arquivo variables.less: @baseFontSize e @baseLineHeight. O primeiro é o tamanho da fonte base usado e o segundo é a altura da linha base.

Usamos essas variáveis, e um pouco de matemática, para criar as margens, preenchimentos e alturas de linha de todos os nossos tipos e muito mais.

Código Snippets de código inline e de bloco

Inline

Envolva trechos de código inline com <code>. 

Por exemplo, <code>section</code> deve ser agrupado como inline.

Bloco básico

Use <pre> para várias linhas de código. Certifique-se de escapar de quaisquer colchetes angulares no código para uma renderização adequada. 

<p>Exemplo de texto aqui...</p>
<pre>
    &lt;p&gt;Exemplo de texto aqui...&lt;/p&gt;
</pre>

Observação: Certifique-se de manter o código dentro das tags <pre> o mais próximo possível da esquerda; ele renderizará todas as guias.

Você pode opcionalmente adicionar a classe .pre-scrollable que definirá uma altura máxima de 350px e fornecerá uma barra de rolagem no y-axis.

Google Prettify

Pegue o mesmo elemento <pre> e adicione duas classes opcionais para renderização aprimorada. 

<p>Exemplo de texto aqui...</p>
<pre class="prettyprint linenums">
    &lt;p&gt;
        Exemplo de texto aqui...
    &lt;/p&gt;
</pre>

Baixar google-code-prettify e veja o leia-me sobre como usar.

Formulários

HTML e CSS flexíveis

A melhor parte dos formulários no Bootstrap é que todas as suas entradas e controles ficam ótimos, não importa como você os cria em sua marcação. Nenhum HTML supérfluo é necessário, mas fornecemos os padrões para quem precisa.

Layouts mais complicados vêm com classes sucintas e escalonáveis para fácil estilização e vinculação de eventos, para que você esteja coberto em cada etapa.

Quatro layouts incluídos

Bootstrap vem com suporte para quatro tipos de layouts de formulário:

  • Vertical (padrão)
  • Busca
  • Inline
  • Horizontal

Diferentes tipos de layouts de formulário exigem algumas alterações na marcação, mas os próprios controles permanecem e se comportam da mesma forma.

Estados de controle e muito mais

Os formulários do Bootstrap incluem estilos para todos os controles básicos do formulário, como entrada, área de texto e seleção esperada. Mas também vem com vários componentes personalizados, como entradas anexadas e pré-anexadas e suporte para listas de caixas de seleção.

Estados como erro, aviso e sucesso são incluídos para cada tipo de controle de formulário. Também estão incluídos estilos para controles desabilitados.

Quatro tipos de formulários

Bootstrap fornece marcação e estilos simples para quatro estilos de formulários web comuns.

Nome Classe Descrição
Vertical (padrão) .form-vertical (not required) Etiquetas empilhadas e alinhadas à esquerda sobre controles
Inline .form-inline Etiqueta alinhada à esquerda e controles de bloco inline para estilo compacto
Search .form-search Entrada de texto extra-arredondada para uma estética de pesquisa típica
Horizontal .form-horizontal Flutue rótulos alinhados à esquerda e à direita na mesma linha dos controles

Refletindo os estilos padrão do WebKit, basta adicionar .form-search para campos de pesquisa extra arredondados.

Formulário inline

As entradas estão no nível do bloco para iniciar. Para .form-inline e .form-horizontal, usamos bloco inline.

    <input type="password" class="input-small" placeholder="Senha">

    <button type="submit" class="btn">
        Login
    </button>

Acrescentar e anexar entradas

Grupos de entrada - com texto anexado ou pré-anexado - fornecem uma maneira fácil de fornecer mais contexto para suas entradas. Ótimos exemplos incluem o sinal @ para nomes de usuário do Bwritec ou $ para finanças.

Checkbox e radios

Até a versão 1.4, o Bootstrap exigia marcação extra em torno das caixas de seleção e dos rádios para empilhá-los. Agora, é uma simples questão de repetir o <label class="checkbox"> que envolve o <input type="checkbox">.

Checkbox e radio inline também são suportados. Basta adicionar .inline para qualquer .checkbox ou .radio e pronto.

Formulários inline e append/prepend

Para usar entradas pré-anexadas ou anexadas em um formulário inline, certifique-se de colocar o .add-on e input na mesma linha, sem espaços.

Texto de ajuda do formulário

Para adicionar texto de ajuda às entradas do formulário, inclua texto de ajuda inline com <span class="help-inline"> ou um bloco de texto de ajuda com <p class="help-block"> após o elemento de entrada.

Botões para ações

Como convenção, os botões só devem ser usados para ações, enquanto os hiperlinks devem ser usados para objetos. Por exemplo, “Download” deve ser um botão, enquanto “atividade recente” deve ser um link.

Os estilos de botão podem ser aplicados a qualquer coisa com a classe .btn aplicada. No entanto, normalmente você desejará aplicá-los apenas em elementos <a> e <button>.

Compatibilidade entre navegadores

O IE9 não remove gradientes de fundo em cantos arredondados, então nós o removemos. Relacionado, IE9 jankifies desativado em elementos button, tornando o texto cinza com uma sombra de texto desagradável que não podemos consertar.

Vários tamanhos

Quer botões maiores ou menores ? Adicionar .btn-large, .btn-small, ou .btn-mini para dois tamanhos adicionais.


Estado desativado

Para botões desativados, adicione a classe .disabled para links e o atributo disabled para elementos <button>.

Link primário Link

Atenção ! Nós usamos .disabled como uma classe utilitária aqui, semelhante á classe .active comum, portanto, nenhum prefixo é necessário.

One class, multiple tags

Use a classe .btn em um elemento <a>, <button>, ou <input>.

Link 
<a class="btn" href="">Link</a>
<button class="btn" type="submit">
    Botão
</button>

<input class="btn" type="button" value="Input">
<input class="btn" type="submit" value="Submit">

Como prática recomendada, tente combinar o elemento com seu contexto para garantir a renderização entre navegadores correspondente. Se você tem um input, use um <input type="submit"> para o seu botão.

Atenção ! As classes de ícones são ecoadas via :after CSS. Nos documentos, mostramos uma cor de fundo vermelho claro ao passar o mouse para destacar o tamanho do ícone.

Construído como um sprite

Em vez de tornar cada ícone uma solicitação extra, nós os compilamos em um sprite - um monte de imagens em um arquivo que usa CSS para posicionar as imagens com background-position. Este é o mesmo método que usamos no bwritec.com.br e funcionou bem para nós.

Todas as classes de ícones são prefixadas com .icon- para namespace e escopo adequados, assim como nossos outros componentes. Isso ajudará a evitar conflitos com outras ferramentas.

Glyphicons nos concedeu o uso do conjunto Halflings em nosso kit de ferramentas de código aberto, desde que forneçamos um link e crédito aqui nos documentos. Por favor, considere fazer o mesmo em seus projetos.

Como usar

Com a v2.0.1, optamos por usar uma tag <i> para todos os nossos ícones, mas eles não possuem classe de caso - apenas um prefixo compartilhado. Para usar, coloque o seguinte código em qualquer lugar: 

<i class="icon-search"></i>

Também existem estilos disponíveis para ícones invertidos (brancos), preparados com uma classe extra: 

<i class="icon-search icon-white"></i>

Existem 120 classes para escolher para seus ícones. Basta adicionar uma tag <i> com as classes certas e pronto. Você pode encontrar a lista completa em sprites.less ou aqui mesmo neste documento.

Casos de Uso

Os ícones são ótimos, mas onde usá-los ? Aqui estão algumas ideias:

  • Como recursos visuais para a navegação da barra lateral
  • Para uma navegação puramente baseada em ícones
  • Para que os botões ajudem a transmitir o significado de uma ação
  • Com links para compartilhar contexto no destino de um usuário

Essencialmente, em qualquer lugar onde você possa colocar uma tag <i>, você pode colocar um ícone.

Exemplos

Use-os em botões, grupos de botões para uma barra de ferramentas, navegação ou entradas de formulário anexadas.
